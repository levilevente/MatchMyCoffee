This directory contains the user interface for **MatchMyCoffee**. It is a modern Single Page Application (SPA) built with React 19, TypeScript, and Vite, providing a coffee e-commerce experience with product browsing, cart management, favorites, and blog content.

## ğŸ— Architecture

The frontend is built as a standalone React application using **Vite** for fast development and bundling. It features:

- **React Router v7** for client-side routing
- **React Bootstrap** for UI components and responsive design
- **Context API** for state management (Cart & Favorites)
- **i18next** for internationalization support
- **Axios** for HTTP requests

The application is designed to consume APIs from:

- **Backend:** Spring Boot (User data, Products, Orders)
- **AI Agent:** Python/LangGraph (Recommendations, Intelligent Chat)

## âœ¨ Features

### Current Implementation

- **Product Catalog:** Browse coffee products with grid layout and pagination
- **Product Details:** Detailed product pages with reviews and ratings
- **Shopping Cart:** Add/remove products, update quantities, and checkout
- **Favorites:** Save favorite products for later
- **Blog:** Browse blog posts and read detailed articles
- **Internationalization:** Multi-language support via i18next
- **Responsive Design:** Mobile-friendly layout using React Bootstrap

### Pages

- `/` - Home page with product catalog
- `/products/:productId` - Individual product details
- `/cart` - Shopping cart overview
- `/cart/checkout` - Checkout page
- `/favorites` - User's favorite products
- `/blog` - Blog posts listing
- `/blog/:blogId` - Individual blog post

## ğŸš€ Getting Started

### Prerequisites

- **Node.js:** v20.19.0 or higher
- **Yarn:** This project uses Yarn for dependency management

### Installation

Navigate to the frontend directory (if you are at the repo root):

```bash
cd frontend
yarn install
```

### Environment Configuration

Copy the `.env-template` file to `.env` and configure your environment variables:

```bash
cp .env-template .env
```

### Running Locally

Start the development server with:

```bash
yarn dev
```

The app will be available at `http://localhost:5173` by default.

> **Note:** This command runs the Frontend only. Currently, the app uses mock data from `DummyProducts.ts`. To interact with real data (Coffees, Orders) or the AI Agent, ensure the Spring Boot and Python services are running. Refer to the Root README for full stack orchestration.

### Key Architectural Decisions

- **Context API:** Used for global state (cart and favorites) to avoid prop drilling
- **Custom Hooks:** Business logic extracted into reusable hooks (`usePagination`, `useProductActions`)
- **TypeScript:** Strong typing throughout for better developer experience and fewer runtime errors
- **Modular Components:** Components are organized by feature and responsibility
- **Mock Data:** Development can proceed independently of backend availability

## ğŸŒ Internationalization

The app uses **i18next** for multi-language support:

- Configuration: `src/i18n.js`
- Translation files: `public/locales/{language}/translation.json`
- Browser language detection enabled
- HTTP backend for loading translations

To add a new language, create a new folder under `public/locales/` with the language code and add a `translation.json` file.

## ğŸ”„ State Management

The app uses **React Context API** for managing global state:

- **CartContext:** Manages shopping cart items, quantities, and cart operations
- **FavoritesContext:** Manages user's favorite products

Both contexts persist data to `localStorage` for a seamless user experience across sessions.
